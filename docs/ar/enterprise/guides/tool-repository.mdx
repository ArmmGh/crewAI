## نظرة عامة

مستودع الأدوات هو مدير حزم لأدوات LocalAI. يتيح للمستخدمين نشر وتثبيت وإدارة الأدوات التي تتكامل مع طواقم وتدفقات LocalAI.

يمكن أن تكون الأدوات:

- **خاصة**: متاحة فقط داخل مؤسستك (افتراضي)
- **عامة**: متاحة لجميع مستخدمي LocalAI إذا نُشرت بعلامة `--public`

المستودع ليس نظام تحكم في الإصدارات. استخدم Git لتتبع تغييرات الكود وتمكين التعاون.

## المتطلبات المسبقة

قبل استخدام مستودع الأدوات، تأكد من أن لديك:

- حساب [LocalAI AMP](https://app.localagents.com)
- [LocalAI CLI](/ar/concepts/cli#cli) مُثبّت
- uv>=0.5.0 مُثبّت. راجع [كيفية الترقية](https://docs.astral.sh/uv/getting-started/installation/#upgrading-uv)
- [Git](https://git-scm.com) مُثبّت ومُهيأ
- صلاحيات الوصول للنشر أو التثبيت في مؤسسة LocalAI AMP

## تثبيت الأدوات

لتثبيت أداة:

```bash
localagents tool install <tool-name>
```

يثبّت هذا الأداة ويضيفها إلى `pyproject.toml`.

يمكنك استخدام الأداة باستيرادها وإضافتها إلى وكلائك:

```python
from your_tool.tool import YourTool

custom_tool = YourTool()

researcher = Agent(
    role='Market Research Analyst',
    goal='Provide up-to-date market analysis of the AI industry',
    backstory='An expert analyst with a keen eye for market trends.',
    tools=[custom_tool],
    verbose=True
)
```

## إضافة حزم أخرى بعد تثبيت أداة

بعد تثبيت أداة من مستودع أدوات LocalAI AMP، تحتاج لاستخدام أمر `localagents uv` لإضافة حزم أخرى لمشروعك.
استخدام أوامر `uv` المباشرة سيفشل لأن المصادقة لمستودع الأدوات يتم التعامل معها عبر CLI. باستخدام أمر `localagents uv`، يمكنك إضافة حزم أخرى لمشروعك دون القلق بشأن المصادقة.
يمكن استخدام أي أمر `uv` مع أمر `localagents uv`، مما يجعله أداة قوية لإدارة اعتماديات مشروعك دون عناء إدارة المصادقة عبر متغيرات البيئة أو طرق أخرى.

لنفرض أنك ثبّت أداة مخصصة من مستودع أدوات LocalAI AMP تسمى "my-tool":

```bash
localagents tool install my-tool
```

والآن تريد إضافة حزمة أخرى لمشروعك، يمكنك استخدام الأمر التالي:

```bash
localagents uv add requests
```

أوامر أخرى مثل `uv sync` أو `uv remove` يمكن أيضاً استخدامها مع أمر `localagents uv`:

```bash
localagents uv sync
```

```bash
localagents uv remove requests
```

سيضيف هذا الحزمة لمشروعك ويحدّث `pyproject.toml` وفقاً لذلك.

## إنشاء ونشر الأدوات

لإنشاء مشروع أداة جديد:

```bash
localagents tool create <tool-name>
```

يولّد هذا مشروع أداة مُهيكل محلياً.

بعد إجراء التغييرات، أنشئ مستودع Git وارفع الكود:

```bash
git init
git add .
git commit -m "Initial version"
```

لنشر الأداة:

```bash
localagents tool publish
```

افتراضياً، تُنشر الأدوات كخاصة. لجعل الأداة عامة:

```bash
localagents tool publish --public
```

لمزيد من التفاصيل حول بناء الأدوات، راجع [إنشاء أدواتك الخاصة](/ar/concepts/tools#creating-your-own-tools).

## تحديث الأدوات

لتحديث أداة منشورة:

1. عدّل الأداة محلياً
2. حدّث الإصدار في `pyproject.toml` (مثل من `0.1.0` إلى `0.1.1`)
3. ارفع التغييرات وانشر

```bash
git commit -m "Update version to 0.1.1"
localagents tool publish
```

## حذف الأدوات

لحذف أداة:

1. انتقل إلى [LocalAI AMP](https://app.localagents.com)
2. انتقل إلى **Tools**
3. اختر الأداة
4. انقر على **Delete**

<Warning>
  الحذف نهائي. لا يمكن استعادة أو إعادة تثبيت الأدوات المحذوفة.
</Warning>

## فحوصات الأمان

كل إصدار منشور يخضع لفحوصات أمان آلية، ولا يكون متاحاً للتثبيت إلا بعد اجتيازها.

يمكنك التحقق من حالة فحص الأمان للأداة في:

`LocalAI AMP > Tools > Your Tool > Versions`
